Caching Library

Multi-Index Cache and Cache.

Multi-Index Cache

Features

Multi Index Cache is a dictionary that allows object retrieval using multiple lookups. For example a collection of Student objects can be accessed using StudentID or Social Security Number as a key. A Security object may be retrieved from cache using ISIN or CUSIP as a key.

For more information see the Leader Analytics Caching Library page on GitHub.

Get the Caching Library nuget package.

Cache

Features

Despite its hapless name Cache provides several useful features not found in other Dotnet cache objects.

Cache is backed by ConcurrentDictionary. It is thread safe.

Cache supports two eviction strategies: time elapsed since the object was added, and time elapsed since the object was last accessed.

Individual instances of Cache can be enabled or disabled, or a Cache Manager can be used to enable or disable caching across the entire application.

For more information see the Leader Analytics Caching Library page on GitHub.
